General Tab

Previous | Next | See Also

The General Tab provides a unified view on the metadata of different types of data sources. It is arranged in a series of panels, each panel showing a specific type of metadata. The type of metadata is indicated by the panel's title.


All metadata are shown by default. Missing panels imply that the related information is not available in the data sources. To better view the data, you can click to hide the metadata you are not interested in. If the label is too long, you can mouse over the label to view the full label as a tooltip.


You can find out more about each metadata panels in the respective sections below:

  • General, Reference, Contact, File Content
  • Sample, Protocol, Instrument Configuration
  • Software, Scan Setting, Data Processing

General

The overall information about the experiment.
For mzML, you will see the ID, accession and version of the experiment.
For PRIDE XML, you will see additional information, such as: title, short label, project name, experiment description and etc.

Reference

Any publication related to this data source. It consists of a full reference field and a PubMed field. For more details, you can click on the hyperlink to go directly to the PubMed website.
In mzML, reference information does not exist.
In PRIDE XML, reference information is optional.

Contact

Contact details for the people who are responsible for the experiment.
In mzML, contact information does not exist.
In PRIDE XML, contact information is required.

File Content

Description of what type of data is stored in the data source. For instance, you can see "mass spectrum", which indicates that the data source contains spectra.

Sample

Additional information on the sample investigated in the experiment. Both mzML and PRIDE XML format contain this information. It generally contains the species, tissue and cell type investigated in the experiment.

Protocol

Detailed description of the experimental steps. It starts with a name for the protocol, followed by a list of steps. Note that, only PRIDE XML contains this information.

Instrument Configuration

Instrument Configuration captures the mass spectrometer model. It contains the following key sections:

  • Instrument Name - Descriptive name of the instrument (make, model, significant customisations)
  • Source - Invariant ion source (e.g. MALDI) information, as a run of name-value pairs
  • Analyzer - Mass analyzer component (e.g. quadrupole, collision cell), ordered so as to reflect the physical order of the described components in the mass spectrometer.
  • Detector - Ion detector information, as a run of name-value pairs.

Software

Software information, for example: the software that produced the peak list. It contains the official name for the software package used.

Scan Setting

Description of the acquisition settings of the instrument prior to the start of the run.

Data Processing

Description of the default processing by which the peak list(s) were generated.
